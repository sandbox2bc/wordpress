이 가이드는 WordPress + WooCommerce + Dokan 환경에서 Advanced Media Offloader 플러그인을 사용해 미디어 파일을 Backblaze B2 클라우드 스토리지로 오프로드하는 전체 설정 방법을 단계별로 안내합니다.

0

사전 준비 — 필요한 것 확인

  • Backblaze B2 계정 (없으면 backblaze.com에서 무료 가입)
  • WordPress 관리자 계정 및 서버의 wp-config.php 편집 권한
  • Advanced Media Offloader 플러그인 (WordPress.org 무료 제공)
  • 서버에 PHP cURL 및 AWS SDK 요구사항 충족 (PHP 7.4+)
✅ WooCommerce / Dokan 호환성: Advanced Media Offloader는 WordPress 핵심 훅을 사용해 URL을 재작성하므로 WooCommerce 상품 이미지, 갤러리, Dokan 벤더 업로드 파일 모두 자동으로 오프로드됩니다.
1

Backblaze B2 — 버킷(Bucket) 생성

  1. backblaze.com에 로그인 후 좌측 메뉴에서 B2 Cloud Storage → Buckets 클릭
  2. Create a Bucket 버튼 클릭
  3. 버킷 이름 입력 (예: mysite-media) — 영문 소문자, 숫자, 하이픈만 사용
  4. Files in Bucket are: → Public 선택 (미디어 파일 공개 접근 필요)
  5. 나머지 옵션(Encryption, Object Lock)은 기본값 유지
  6. Create a Bucket 클릭
💡 버킷 생성 후 확인할 정보: 버킷 목록에서 생성된 버킷을 클릭하면 Endpoint 주소가 표시됩니다. 이 주소를 메모해 두세요.
예: s3.us-west-004.backblazeb2.com

엔드포인트 및 도메인 URL 구조

Endpoint URL (wp-config.php에 입력할 값)
https://s3.리전코드.backblazeb2.com
Domain URL (파일 서빙에 사용)
https://버킷이름.s3.리전코드.backblazeb2.com/

예: 버킷명 mysite-media, 리전 us-west-004인 경우

Endpoint → https://s3.us-west-004.backblazeb2.com
Domain   → https://mysite-media.s3.us-west-004.backblazeb2.com/
2

Backblaze B2 — Application Key (API 키) 생성

  1. Backblaze 대시보드 좌측 메뉴 → Application Keys
  2. Add a New Application Key 클릭
  3. 아래 표대로 설정 후 Create New Key 클릭
설정 항목 입력값 설명
Name of Key WordPress-Offload-Key 식별하기 쉬운 이름 (자유롭게 설정)
Allow access to Bucket(s) Step 1에서 만든 버킷 선택 "All" 대신 특정 버킷만 선택 권장 (보안)
Type of Access Read and Write 업로드 및 읽기 모두 필요
File name prefix 비워두기 플러그인이 경로를 자동 관리
Duration (in seconds) 비워두기 만료 없음 (무기한 유효)
⚠️ 중요: 키 생성 직후 표시되는 keyID와 applicationKey를 반드시 즉시 복사해서 안전한 곳에 저장하세요. 창을 닫으면 Application Key는 다시 확인할 수 없습니다!

복사해야 할 두 가지 값

항목wp-config.php 상수명예시 형태
keyID ADVMO_BACKBLAZE_B2_KEY 004abc123def...
applicationKey ADVMO_BACKBLAZE_B2_SECRET K004XXXXXXXXXXXXXXXXXXXX
3

Advanced Media Offloader 플러그인 설치 및 활성화

  1. WordPress 관리자 → 플러그인 → 새 플러그인 추가
  2. 검색창에 Advanced Media Offloader 입력
  3. 지금 설치 클릭 → 설치 완료 후 활성화 클릭
  4. 활성화 후 좌측 메뉴에 Media Offloader 항목이 생성됨
💡 수동 설치: WordPress.org에서 ZIP 파일을 다운로드한 경우 플러그인 → 새 플러그인 추가 → 플러그인 업로드에서 ZIP 파일을 업로드하여 설치할 수 있습니다.
4

wp-config.php에 Backblaze B2 인증 정보 추가

플러그인의 보안 정책상 클라우드 인증 정보는 데이터베이스가 아닌 wp-config.php 파일에 저장해야 합니다.

  1. FTP/SFTP 또는 호스팅 파일 관리자로 서버의 wp-config.php 파일을 엽니다
    (WordPress 루트 디렉터리에 위치)
  2. 아래 코드를 찾아 그 바로 위에 B2 설정 코드를 붙여넣습니다:
👆 이 줄 바로 위에 아래 코드를 삽입하세요: /* That's all, stop editing! Happy publishing. */
wp-config.php — Backblaze B2 설정
// ===== Advanced Media Offloader - Backblaze B2 설정 =====
define('ADVMO_BACKBLAZE_B2_KEY',      'your-key-id');           // keyID
define('ADVMO_BACKBLAZE_B2_SECRET',   'your-application-key');  // applicationKey
define('ADVMO_BACKBLAZE_B2_BUCKET',   'your-bucket-name');      // 버킷 이름
define('ADVMO_BACKBLAZE_B2_REGION',   'us-west-004');           // 버킷 리전
define('ADVMO_BACKBLAZE_B2_DOMAIN',   'https://your-bucket-name.s3.us-west-004.backblazeb2.com/');
define('ADVMO_BACKBLAZE_B2_ENDPOINT', 'https://s3.us-west-004.backblazeb2.com');
// ========================================================
💡 각 값 입력 방법:
• KEY → Step 2에서 복사한 keyID
• SECRET → Step 2에서 복사한 applicationKey
• BUCKET → Step 1에서 생성한 버킷 이름
• REGION → 버킷 상세 페이지의 Endpoint에서 확인 (예: us-west-004)
• DOMAIN → https://버킷명.s3.리전.backblazeb2.com/ (끝에 / 포함)
• ENDPOINT → https://s3.리전.backblazeb2.com (끝에 / 없음)
⚠️ 보안 주의: wp-config.php 파일을 공개 저장소(GitHub 등)에 업로드하거나 공유하지 마세요. API 키가 노출되면 즉시 Backblaze에서 해당 키를 삭제하고 재발급해야 합니다.

파일 저장 후 서버에 업로드하세요.

5

WordPress 대시보드에서 플러그인 설정 및 연결 테스트

  1. WordPress 관리자 → Media Offloader → General Settings
  2. Cloud Provider 항목에서 Backblaze B2 선택
  3. Test Connection 버튼 클릭 → 성공 메시지 확인
⚠️ 연결 실패 시 확인 사항:
• wp-config.php의 KEY / SECRET 오타 여부
• ENDPOINT URL에 https:// 포함 여부
• DOMAIN URL 끝에 / 포함 여부
• Backblaze에서 해당 버킷의 Access가 Public인지 확인

주요 설정 옵션 설명

설정 항목 권장값 설명
Offload New Uploads 활성화 새로 업로드되는 미디어 자동 오프로드
Retain Local Files 초기: 활성화 오프로드 후 서버 원본 파일 유지 (안전하게 시작)
Mirror Delete 선택 사항 WordPress에서 파일 삭제 시 B2에서도 삭제
File Versioning 선택 사항 파일 경로에 타임스탬프 추가 (캐시 문제 방지)
Custom Path Prefix 비워두기 또는 wp-content/uploads B2 버킷 내 저장 경로 접두어
💡 초기 설정 권장: 처음에는 Retain Local Files를 활성화해서 로컬 파일을 유지하면서 테스트하세요. 정상 작동 확인 후 로컬 파일 삭제를 고려하세요.
6

기존 미디어 파일 일괄 오프로드 (Bulk Offload)

이미 업로드된 파일들을 B2로 이전하려면 Bulk Offload 기능을 사용합니다.

  1. WordPress 관리자 → Media Offloader → Media Overview
  2. 현재 로컬에만 있는 파일 수 확인
  3. Bulk Offload (또는 Start Offloading) 버튼 클릭
  4. 무료 버전: 배치당 50개 파일씩 처리 → 완료될 때까지 페이지 유지
💡 대용량 라이브러리: 파일이 수천 개 이상이면 배치 처리가 여러 번 반복됩니다. 브라우저 탭을 닫지 말고 완료될 때까지 기다리세요. 중단 시 Cancel 버튼으로 중지할 수 있습니다.
✅ WooCommerce 상품 이미지: 상품 대표 이미지, 갤러리 이미지, WooCommerce 전용 썸네일 크기 모두 자동으로 함께 오프로드됩니다.
7

Dokan 멀티벤더 환경 추가 확인 사항

Dokan에서 벤더(판매자)가 상품을 등록할 때 업로드하는 이미지도 WordPress 미디어 라이브러리를 통해 처리되므로 Advanced Media Offloader가 자동으로 적용됩니다. 아래 항목을 추가로 확인하세요.

확인 항목 확인 방법 상태
벤더 상품 이미지 URL 벤더 상품 페이지 → 이미지 오른쪽 클릭 → URL 확인 B2 도메인으로 시작해야 정상
벤더 스토어 배너/로고 Dokan 스토어 페이지 확인 wp-content/uploads 경유 URL이면 오프로드 됨
디지털 다운로드 파일 WooCommerce → 상품 → 다운로드 파일 URL 확인 Private 파일은 현재 무료 버전 미지원
캐시 플러그인 캐시 전체 삭제 후 확인 이전 URL이 캐시될 수 있으므로 반드시 삭제
⚠️ 디지털 다운로드 파일 주의: WooCommerce의 Downloadable Products 파일(구매 후 다운로드 제공 파일)을 B2로 오프로드하면 URL이 공개됩니다. 유료 파일은 반드시 Retain Local Files 정책을 사용하거나 오프로드 대상에서 제외하세요.
8

(선택) Cloudflare CDN 연동으로 무료 트래픽 사용

Backblaze는 Cloudflare Bandwidth Alliance 파트너입니다. Cloudflare를 CDN으로 설정하면 B2 ↔ Cloudflare 간 트래픽이 무료입니다.

방문자
→
Cloudflare CDN
→
Backblaze B2
→
이그레스 비용 0원
  1. Cloudflare 대시보드 → 도메인 선택 → DNS
  2. CNAME 레코드 추가: media.yourdomain.com → 버킷명.s3.리전.backblazeb2.com
  3. Proxy status를 Proxied (주황색 구름)으로 설정
  4. wp-config.php의 ADVMO_BACKBLAZE_B2_DOMAIN을 커스텀 도메인으로 변경:
    https://media.yourdomain.com/
✅ 결과: 이미지 URL이 media.yourdomain.com으로 제공되어 브랜드 일관성을 유지하면서 Cloudflare 글로벌 CDN 가속을 무료로 사용할 수 있습니다.

최종 정상 작동 확인 체크리스트

  • 새 이미지를 미디어 라이브러리에 업로드 → B2 버킷에서 파일 확인
  • 미디어 라이브러리의 이미지 URL이 B2 도메인으로 시작하는지 확인
  • WooCommerce 상품 페이지 이미지 정상 로딩 확인
  • Dokan 벤더 스토어 이미지 정상 로딩 확인
  • 워드프레스 캐시 플러그인의 캐시 삭제 후 재확인
  • Media Offloader → Media Overview에서 에러 항목 없는지 확인
